Про SmartInvent

SmartInvent — це україномовна інформаційна система для автоматизації інвентаризації товарів у складських приміщеннях. Система поєднує мобільний додаток та серверну частину з підтримкою QR-кодів для точного обліку товарів.

Актуальність
Традиційні методи інвентаризації є застарілими, трудомісткими та схильними до помилок. SmartInvent вирішує ці проблеми, забезпечуючи ефективність обліку, адаптацію до локальних умов та зменшення людського фактору.

Мета роботи
Розробка серверної частини системи для інвентаризації з підтримкою генерації та сканування QR-кодів, захисту даних і взаємодії з мобільним клієнтом через REST API.

Методи дослідження
Аналіз сучасних систем обліку, проєктування архітектури, реалізація REST API, моделювання бази даних та впровадження безпечної автентифікації та авторизації.

Очікувані результати
Система дозволить підприємствам автоматизувати облік товарів, скоротити витрати на інвентаризацію, зменшити навантаження на персонал та підвищити точність ведення складських операцій.

Технології, які ми використовуємо
Система SmartInvent побудована на сучасних, надійних та перевірених технологіях, які забезпечують стабільну роботу, безпеку та масштабованість додатка.
Java 17

Основна мова програмування серверної частини. Java 17 забезпечує стабільність, високу продуктивність та довгострокову підтримку (LTS).

Spring Boot

Основний фреймворк для розробки бекенду. Дозволяє швидко створювати масштабовані REST API, має вбудований Tomcat для запуску без зовнішнього сервера.

Spring Data JPA + Hibernate

Потужний механізм для роботи з базою даних. Спрощує управління сутностями, мінімізує рутинний код та забезпечує гнучке формування запитів.

PostgreSQL

Надійна реляційна СУБД з підтримкою транзакцій ACID. Забезпечує збереження, масштабованість та швидкий доступ до даних у будь-яких умовах.

Spring Security + JWT

Забезпечення безпеки реалізовано через автентифікацію користувачів, перевірку токенів та обмеження доступу на основі ролей. Гарантує конфіденційність даних.

REST API

Комунікація між додатком і сервером побудована на основі REST API (HTTP-запити типу GET, POST, PUT, DELETE) для простої та зрозумілої інтеграції.

Gradle

Інструмент для збірки проєкту, керування залежностями та автоматизації розгортання. Використовується як на Android, так і на сервері.

IntelliJ IDEA

Основне середовище розробки. Забезпечує зручну роботу з Spring Boot, Git, Gradle та надає потужні інструменти для налагодження коду.

Серверна частина

Бекенд забезпечує стабільну роботу системи, обробку бізнес-логіки та взаємодію з базою даних. Реалізовано на основі Spring Boot із використанням PostgreSQL для надійного зберігання даних.

REST API

Сервер надає REST API для обміну даними з мобільним додатком. Всі запити обробляються за допомогою контролерів, сервісів та репозиторіїв згідно з архітектурою MVC.

База даних PostgreSQL

Вся інформація про товари, склади, користувачів та інвентаризацію зберігається у структурованій реляційній базі даних. Кожна компанія має ізольовану базу, що підвищує безпеку та масштабованість.

Аутентифікація та ролі

Система підтримує аутентифікацію з розподілом ролей: адміністратор та співробітник. Це дозволяє гнучко керувати правами доступу до функціоналу системи через рольову модель безпеки.

QR-коди в SmartInvent

Повноцінна підтримка QR-кодів для автоматизованої ідентифікації товарів у процесі інвентаризації на промислових підприємствах.

Створення та генерація QR-кодів

Кожен товар має унікальний productWorkId, який кодується у вигляді QR-коду. Його можна згенерувати автоматично або відсканувати з уже наявного. Код відображається на екрані створення товару та може бути збережений, надрукований або надісланий.

Сканування та робота з кодами

Сканери інтегровані у різні частини додатку: швидкий пошук у списку товарів, відкриття картки товару, сканування під час інвентаризації. Застосовується бібліотека ZXing для стабільної роботи навіть у складних умовах.

Перевірка, унікальність та зручність

QR-код містить лише текстову форму productWorkId без додаткових даних. Система перевіряє унікальність на сервері, інформує про дублікати, дозволяє редагування лише після підтвердження та забезпечує простий, швидкий і надійний облік.
